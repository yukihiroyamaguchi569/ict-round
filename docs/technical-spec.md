# ICTラウンドアプリ「らんちゃん」 技術仕様書

**文書番号:** ICT-TS-001  
**対象者:** 情報システム部門・委員会担当者  
**作成日:** 2026年4月  

---

## 1. システム概要

| 項目 | 内容 |
|------|------|
| システム名 | ICTラウンドアプリ「らんちゃん」（ICN感染対策ラウンド） |
| 種別 | Webアプリケーション（Progressive Web App / PWA） |
| 目的 | 病院感染対策ラウンドの記録・報告書作成の電子化・効率化 |
| 利用者 | 感染対策担当看護師（ICN）、ICTメンバー |
| データ保管先 | **端末内のみ**（外部サーバー・クラウドへの送信なし） |

---

## 2. アーキテクチャ

### 2.1 システム構成図

```
[利用者の端末]
  ├── Webブラウザ（Safari / Chrome / Edge）
  │     ├── アプリ本体（HTML / CSS / JavaScript）
  │     ├── Service Worker（オフラインキャッシュ）
  │     └── Web Storage（テーマ設定のみ）
  │
  └── [院内ファイルサーバー または CDN]
        └── 静的ファイル配信（HTML / CSS / JS / 画像）
```

### 2.2 主要技術スタック

| 分類 | 技術 | バージョン |
|------|------|-----------|
| フレームワーク | React | 19.x |
| 言語 | TypeScript | 5.9.x |
| ビルドツール | Vite | 8.x |
| CSSフレームワーク | Tailwind CSS | 4.x |
| Word生成ライブラリ | docx | 9.x |
| ファイル保存 | file-saver | 2.x |

### 2.3 バックエンド・インフラ
- **バックエンドサーバー：なし**
- **データベース：なし**
- **外部API：なし**
- 静的ファイル（HTML/CSS/JS/画像）の配信のみ

## 3. データの取り扱い

### 3.1 入力データの種類

| データ種別 | 内容 | 保存先 | 保存期間 |
|-----------|------|--------|---------|
| 担当者名 | ICNの氏名（任意の文字列） | メモリ（React state） | セッション中のみ |
| 病棟名 | 対象病棟名（任意） | メモリ（React state） | セッション中のみ |
| チェックリスト評価 | 22項目のA/B/C評価 | メモリ（React state） | セッション中のみ |
| 写真データ | 現場写真（JPEG、base64形式） | メモリ（React state） | セッション中のみ |
| 総評テキスト | フリーテキスト | メモリ（React state） | セッション中のみ |
| テーマ設定 | UIカラーテーマの選択 | localStorage | 端末に永続保存 |

> **「セッション中のみ」** = ブラウザページを閉じるか再読み込みすると消去されます。

### 3.2 患者情報の取り扱い
- **本アプリは患者個人情報の入力フォームを持ちません**
- 入力項目は「担当者名」「病棟名」のみです
- 写真撮影機能はありますが、患者情報の入力欄は設けていません

### 3.3 データの外部送信
- **ラウンドデータ（チェック結果・写真・総評）は外部サーバーへ一切送信されません**
- アプリのコード・フォント取得を除き、外部ネットワーク通信は行いません

### 3.4 レポート出力
- 報告書は `.docx` ファイルとして生成されます
- 出力先は以下のいずれかです：
  - 端末のローカルストレージ（ダウンロードフォルダ等）
  - OS標準の共有シート（メール・Teamsへの添付など）
- ファイル生成はブラウザ内（クライアントサイド）で行われ、外部サーバーへの転送はありません

## 4. 通信・ネットワーク

### 4.1 外部通信の一覧

| 通信先 | 目的 | 通信タイミング | 含まれるデータ |
|--------|------|----------------|---------------|
| `fonts.googleapis.com` | フォントCSS取得 | アプリ起動時（初回/キャッシュ切れ時） | なし（GETリクエストのみ） |
| `fonts.gstatic.com` | フォントファイル取得 | アプリ起動時（初回/キャッシュ切れ時） | なし（GETリクエストのみ） |

> **院内イントラネット配置の場合：** ホスティングサーバーをイントラネット内に置き、フォントを静的ファイルに同梱すれば、外部通信をゼロにすることが可能です。

### 4.2 オフライン動作
Service Worker によりアプリシェルをキャッシュしています。

| ネットワーク状態 | 利用可否 |
|----------------|---------|
| オンライン（初回） | 全機能利用可 |
| オフライン（2回目以降） | 全機能利用可（フォントのみ未キャッシュの場合は代替フォントで表示） |

## 5. セキュリティ

### 5.1 データ保護
- ラウンドデータはブラウザのメモリ（React state）にのみ保持されます
- メモリ上のデータはセッション終了と同時に消去されます
- データをサーバーに送信・保存しないため、サーバー側でのデータ漏洩リスクはありません

### 5.2 認証・アクセス制御
- 本アプリ自体にはログイン認証機能を持ちません
- アクセス制御が必要な場合は、ホスティングサーバー側でのIP制限・Basic認証等の実装を推奨します

### 5.3 入力検証
- フォーム入力はフロントエンドで検証を行います
- 外部APIへの接続がないため、SQLインジェクション等のサーバーサイド攻撃リスクはありません

### 5.4 コード実行
- 本アプリはサードパーティのスクリプトを動的読み込みしません
- 依存ライブラリはビルド時に静的バンドルされます

## 6. 動作要件

### 6.1 端末・ブラウザ要件

| 項目 | 要件 |
|------|------|
| OS | iOS 15以降 / Android 9以降 / Windows 10以降 / macOS 12以降 |
| ブラウザ | Safari 15以降 / Chrome 90以降 / Edge 90以降 |
| カメラ | 写真機能を使用する場合に必要（端末内蔵カメラまたはWebカメラ） |
| ストレージ | ブラウザキャッシュ用に約5MB以上の空き容量 |

### 6.2 推奨環境
- スマートフォン（iOS / Android）での利用を主な用途として設計
- タブレットおよびPCでも動作確認済み

## 7. ホスティング要件

### 7.1 配信方式
- 静的ファイル（HTML / CSS / JavaScript / 画像）の配信のみ
- PHPやNode.js等のサーバーサイド処理は不要

### 7.2 必要なサーバー機能

| 要件 | 内容 |
|------|------|
| 静的ファイル配信 | Nginx / Apache / IIS など、静的ファイルを配信できるWebサーバー |
| HTTPS | Service Worker の動作にはHTTPS必須（`localhost` は例外） |
| MIMEタイプ | `.js` に `application/javascript`、`.json` に `application/json` を設定 |

### 7.3 院内イントラネット配置について
外部ネットワークへの接続を制限したい場合、以下の対応で完全イントラネット運用が可能です：

1. ホスティングサーバーを院内イントラネット内に設置
2. Google Fonts を Webフォントからシステムフォント（またはローカルフォント）に切り替え
3. 端末からアプリURLへのアクセスを院内Wi-Fiに限定

## 9. ビルド・デプロイ

### 9.1 ビルド手順
```bash
npm install       # 依存関係のインストール
npm run build     # 本番用静的ファイルの生成（dist/ フォルダに出力）
```

### 9.2 デプロイ手順
1. `npm run build` を実行し `dist/` フォルダを生成
2. `dist/` フォルダの内容をWebサーバーの公開ディレクトリに配置
3. WebサーバーのHTTPS設定を確認

### 9.3 更新手順
アプリの更新は静的ファイルの差し替えのみで完了します。データベースのマイグレーション等は不要です。
